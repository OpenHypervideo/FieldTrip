Field Trip is an interactive documentary film about Tempelhof Field, a former airfield now being used as one of the biggest urban parks in the world. For some, the 300 hectares of open space are full of possibilities; for others, it is a place of oppression and of dark episodes in German history. For others still, it has become a home far away from home. Field Trip tells their stories.

Through interactive documentary films and the inclusion of people on site, Field Trip has become a vibrant reflection of one of the most exciting places in Berlin.

The project was exclusively pre-released by the German newspaper Tagesspiegel on the occasion of the 70th anniversary of the end of the Berlin Blockade. Field Trip is a ronjafilm production.

Following the pre-release, the Field Trip team has made a large part of the materials (films, illustrations, software, design) available to the public. Most of the materials are published under free licences.

FAQ

Field Trip is a web-based interactive documentary film. It is the first “web documentary” format which not only relies fully on open source technologies, but also – whenever possible – on open licences for the content produced. We also call it an “open source documentary”.

What is a “web documentary”?

A web documentary is a documentary film created with the help of web technologies such as HTML, CSS and Javascript and played in a browser.

In the case of Field Trip, a web documentary was absolutely essential. Without these technologies, the non-linear narrative concept would not work. Just like the web is based on hypertext links, Field Trip is based on hypervideo links which allow the user to navigate interactively through a network of interconnected narrative threads.

What is an “open source documentary”?

The idea behind Field Trip is to create a permanently open, changeable project based on open web technologies. This means that all interactive options, visualisations, transitions, effects and everything else apart from the raw material is implemented through small, re-usable snippets of code. Instead of conventional post-production software, we are using HTML, Javascript and CSS wherever this is technologically feasible. In addition, we are making these code snippets available in an open repository under MIT and GPL v3 licenses.

Which technologies are being used?

We are using the „FrameTrail Open Hypervideo“ technology for Field Trip. This allows us to edit and link all project components directly in a browser. FrameTrail is an open source software (dual-licensed under MIT and GPL v3 licences) and was developed exclusively with open web technologies and standards. To enable us to edit cuts, transitions and effects, we will integrate the VideoContext library into FrameTrail. VideoContext is developed by BBC Research & Development in the context of “Object-based Media”.

How does the Code Snippet Repository work?

The Code Snippet Repository makes the technical components of the project available as re-usable pieces of code. These could be a small animation, a complex effect or simply a button. The idea is to create independent components which can be re-used in other projects (dissociated from the actual content). To make this as easy as possible, the repository offers both a searchable website and an open interface through which individual code snippets can be retrieved or newly added. The repository is integrated in FrameTrail through the same interface to allow users to insert existing components into videos via simple “drag and drop” as well as to share new components with other users.

How is the content licenced?

While all code is published under open source licences, the conditions for content (e.g. video material, photographs, texts) can vary: We normally use the “Creative Commons Attribution-ShareAlike 4.0 International License”, which allows the free use, editing and distribution (including commercially) of content, as long as the authors are correctly named and the works are passed on under the same conditions. However, in the case of archival material or content obtained through other channels, we do not always have control over the licences. Any restrictions of this kind are clearly communicated in the list of all materials and licences in our Media Repository.
